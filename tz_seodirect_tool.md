## 1. Общее описание

### Что это

Веб-приложение для команды специалистов по поисковому маркетингу. Автоматизирует рутину при работе с клиентами: сбор данных, генерацию стратегий, семантики, объявлений, мета-тегов и OG-тегов. Инструмент усиливает специалиста — не заменяет. Все результаты редактируемы.

### Ключевой принцип

Всё что сгенерировал ИИ — можно и нужно редактировать. Инструмент готовит материал, специалист принимает решения.

### Для кого

Команда: руководитель + специалисты по рекламе и SEO. Развёрнуто на VPS, доступно через браузер.

## 4. Авторизация и роли

### Super Admin

Данные хранятся в `.env` файле на сервере:
```
SUPER_ADMIN_LOGIN=admin
SUPER_ADMIN_PASSWORD=хэш_bcrypt
SUPER_ADMIN_EMAIL=email@domain.ru
```

Super Admin всегда может войти независимо от состояния БД. Создаётся при первом запуске через `docker-compose exec backend python init_superadmin.py`.

### Роли пользователей

| Роль | Описание | Права |
|------|----------|-------|
| `super_admin` | Владелец системы | Всё включая настройки API и управление пользователями |
| `admin` | Руководитель команды | Все проекты, все модули, управление специалистами |
| `specialist` | Специалист | Свои проекты + назначенные, все модули |
| `viewer` | Наблюдатель | Только просмотр назначенных проектов |

### Страница авторизации

- Логин + пароль
- JWT токены: access (15 мин) + refresh (30 дней)
- Refresh автоматический, пользователь не замечает
- «Запомнить меня» — продлевает refresh до 90 дней
- Защита: rate limiting 5 попыток / 15 минут с одного IP
- Нет регистрации — пользователей создаёт только admin/super_admin

### Управление пользователями (только admin+)

- Создать пользователя (логин, email, пароль, роль)
- Деактивировать (не удалять — сохраняем историю)
- Сбросить пароль
- Назначить на проект

## 7. Модуль: Парсинг сайта

### Принципы работы

1. Всегда скачивать и уважать `robots.txt`
2. Искать `sitemap.xml` в стандартных местах + в `robots.txt`
3. Соблюдать `Crawl-delay` из `robots.txt` (или дефолт из настроек)
4. Не парсить запрещённые пути из `robots.txt`
5. Честный `User-Agent` из настроек системы

### Алгоритм парсинга

```
1. GET /robots.txt
   → Парсим Disallow, Allow, Crawl-delay, Sitemap

2. Ищем sitemap:
   → Из robots.txt
   → /sitemap.xml
   → /sitemap_index.xml
   → /sitemap.xml.gz
   Если sitemap_index — рекурсивно обходим все дочерние

3. Из sitemap получаем список URL:
   → url, lastmod, changefreq, priority

4. Если sitemap нет — краулим с главной страницы
   → Собираем внутренние ссылки
   → Максимум страниц из настроек

5. Для каждой разрешённой страницы:
   → GET запрос с Crawl-delay
   → Парсим: title, description, H1, H2-H6,
             canonical, og:*, robots meta,
             количество слов, наличие изображений,
             внутренние и внешние ссылки

6. Сохраняем в БД, обновляем прогресс
```

### Что собираем с каждой страницы

```python
PageData:
  url: str
  status_code: int
  title: str | None
  description: str | None
  h1: str | None
  h2_list: list[str]
  canonical: str | None
  og_title: str | None
  og_description: str | None
  og_image: str | None
  og_type: str | None
  robots_meta: str | None      # noindex, nofollow и т.д.
  word_count: int
  internal_links: list[str]
  external_links: list[str]
  images_without_alt: int
  load_time_ms: int
  last_modified: str | None    # из sitemap
  priority: float | None       # из sitemap
```

### Итоговый отчёт парсинга

После парсинга специалист видит:
- Общая структура сайта (дерево URL)
- Страницы без title / без description / без H1
- Страницы с дублирующимися title/description
- Страницы без OG-тегов
- Orphan pages (нет входящих внутренних ссылок)
- Страницы с noindex
- Медленные страницы (> 3 сек)

## 8. Модуль: Яндекс Директ

### Шаг 1 — Генерация стратегии

После заполнения брифа специалист нажимает «Сгенерировать стратегию Директа».

Claude получает:
- Данные брифа
- Данные парсинга сайта (УТП, структура, контент)
- Инструкцию (системный промпт)

Генерирует:
- Анализ объекта и УТП
- Структуру кампаний (список с приоритетами)
- Геотаргетинг с логикой по каждому городу
- Сезонный медиаплан

Всё редактируемо прямо в интерфейсе.

### Шаг 2 — Семантическое ядро

**Генерация ключей:**
- Claude генерирует ключевые фразы по кампаниям и группам
- Указывает температуру (горячие / тёплые / холодные)

**Проверка частотностей:**
- Каждый ключ отправляется в Wordstat API
- Возвращается частотность + динамика по месяцам
- Ключи ниже порога (настраивается, дефолт 50/мес) помечаются серым

**Расширение через Topvisor:**
- `tasks/collect` — подбор похожих ключей
- `tasks/volumes` — массовая проверка частот

**Интерфейс таблицы ключей:**

| Фраза | Частота | Сезон | Группа | Кампания | Действие |
|-------|---------|-------|--------|----------|---------|
| гостевой дом партенит | 1 240 | 📈 Лето | Гео Партенит | Кампания 2 | ✏️ 🗑️ |

- Фильтрация по кампании, группе, частоте, температуре
- Массовое удаление / перемещение между группами
- Добавление ключа вручную
- Кнопка «Расширить» — Claude предлагает ещё ключи по выбранной группе

### Шаг 3 — Минус-слова

- Claude генерирует минус-слова по блокам (общие / по кампаниям)
- Таблица с возможностью добавить / удалить каждое слово
- Импорт из текстового поля (построчно)
- Проверка — нет ли минус-слов которые пересекаются с ключами

### Шаг 4 — Объявления

Для каждой группы объявлений:

- Claude генерирует 2–3 варианта (A/B)
- Счётчик символов в реальном времени (56/30/30/81)
- Подсветка если превышен лимит
- Редактор прямо в интерфейсе
- Кнопка «Перегенерировать» для любого варианта
- Статус: черновик / готово / на проверке

**Поля объявления:**
```

```

**Быстрые ссылки (единые для кампании):**

| [              ] | [              ] |

### Шаг 5 — Медиаплан

Таблица с прогнозом бюджета по месяцам:

| Месяц | % | Бюджет ₽ | Прогноз кликов | Прогноз заявок | CPA |

## 9. Модуль: SEO

### Семантическое ядро для SEO

Отличие от Директа: ключи группируются не по кампаниям, а **по страницам сайта**.

Источники ключей:
- Topvisor `keywords/collect` — подбор по теме
- Topvisor `snapshots` — что используют конкуренты
- Wordstat — частотности и подсказки
- Claude — дополнение и расширение

**Кластеризация:**
- Topvisor API `claster` — автоматическая группировка
- Результат: каждому кластеру назначается целевая страница
- Специалист подтверждает или меняет назначение

### Мета-теги

Для каждой страницы сайта (из парсинга):

```
URL страницы: /rooms/deluxe
Текущий title: [Номера — Partenitos]         ⚠️ Слишком короткий
Текущий description: —                        ❌ Отсутствует

Рекомендуемый title:
[Номера Deluxe в Партените · Вид на море — Partenitos]  42/60 ✅

Рекомендуемый description:
[Просторные номера Deluxe 21–28 м² с видом на Черное море.
Завтрак включён. Бронируйте на официальном сайте]       118/160 ✅

Ключевые слова в title: [✅ партенит] [✅ номера] [⚠️ добавить: море]
```

Кнопка «Сгенерировать» — Claude пишет title/description на основе контента страницы + ключей кластера.

### ТЗ для копирайтера

Claude генерирует задание для каждой страницы:
- Целевые ключи (основной + LSI)
- Рекомендуемый объём текста
- Структура заголовков (H1, H2...)
- На что сделать акцент
- Что не писать (минус-темы)

Экспорт в DOCX или текст.

### Технический SEO чеклист

На основе данных парсинга — автоматически:

```
Индексация:
  [✅] robots.txt найден и корректен
  [✅] sitemap.xml найден
  [⚠️] 3 страницы с noindex — проверьте намеренность
  [❌] Страница /contacts не в sitemap

Мета-теги:
  [❌] 12 страниц без title
  [❌] 8 страниц без description
  [⚠️] 3 дублирующихся title

Структура:
  [⚠️] 5 страниц без H1
  [❌] 2 страницы с несколькими H1
  [✅] Глубина вложенности: макс. 3 уровня

Производительность:
  [⚠️] 4 страницы загружаются > 3 сек
  [✅] Мобильная версия определена

Изображения:
  [❌] 24 изображения без alt-атрибута
```

### Мониторинг позиций (через Topvisor)

- Добавить ключи в проект Topvisor через API
- Видеть позиции прямо в инструменте
- График динамики позиций

## 12. Модуль: Экспорт

### Форматы экспорта

| Что экспортируем | Формат | Для кого |
|-----------------|--------|---------|
| Стратегия Директа | Markdown / PDF | Для клиента / архив |
| Семантика Директа | XLS (Директ Коммандер) | Загрузка в Директ |
| Семантика SEO | XLSX | Для работы |
| Мета-теги | XLSX | Для разработчика |
| OG-теги | HTML код | Для разработчика |
| ТЗ копирайтеру | DOCX | Для копирайтера |
| Медиаплан | XLSX | Для клиента |

### XLS для Директ Коммандера

Ключевой экспорт. Формат точно соответствует шаблону Яндекс Директ Коммандера.

Листы:
- `Campaigns` — список кампаний
- `AdGroups` — группы объявлений
- `Ads` — объявления
- `Keywords` — ключевые фразы
- `NegativeKeywords` — минус-слова

**Специалист перед экспортом видит:**
```
Готово к выгрузке:
  Кампаний:        16
  Групп:           47
  Объявлений:      94
  Ключевых фраз:   283
  Минус-слов:      214

  ⚠️ 3 объявления не прошли проверку символов — исправьте перед экспортом

  [Скачать XLS]
```

### PDF стратегия для клиента

Красиво оформленный документ:
- Обложка с названием проекта и датой
- Анализ объекта и УТП
- Структура кампаний
- Семантика с частотностями
- Медиаплан
- Рекомендации

Логотип и цвета — настраиваются в настройках (white label под агентство).

## 13. API интеграции

### Claude API (Anthropic)

```
Endpoint: https://api.anthropic.com/v1/messages
Модель:   claude-sonnet-4-20250514
Auth:     X-API-Key header

Использование:
- Анализ брифа и уточняющие вопросы
- Генерация семантического ядра
- Написание объявлений
- Генерация мета-тегов и OG
- ТЗ для копирайтера
- Ответы на правки специалиста
```

Системные промпты хранятся в БД — можно редактировать через `/settings/prompts` (только super_admin).

### Яндекс Wordstat API

```
Endpoint: https://api.wordstat.yandex.net
Auth:     Bearer OAuth токен

Методы:
  POST /v1/topRequests    — частотность + похожие запросы
  POST /v1/dynamics       — динамика по неделям (сезонность)
  POST /v1/regions        — распределение по регионам

Квота: учитываем, батчим запросы через очередь Celery
```

### Topvisor API

```
Endpoint: https://api.topvisor.com/v2
Auth:     API ключ в заголовке

Сервисы:
  keywords_2/collect      — подбор ключевых слов
  keywords_2/volumes      — массовая проверка частот
  keywords_2/claster      — кластеризация
  snapshots_2             — снимки выдачи (конкуренты)
  positions_2             — мониторинг позиций
  audit_2                 — технический аудит
  projects_2              — создание/управление проектами
```

### Яндекс Метрика API

```
Endpoint: https://api-metrika.yandex.net
Auth:     OAuth токен пользователя

Методы:
  stat/v1/data            — получение статистики
  management/v1/counters  — список счётчиков
  management/v1/goals     — цели
```

### Яндекс Директ API (для будущего)

```
Endpoint: https://api.direct.yandex.com/json/v5
Auth:     OAuth токен

Планируется:
  campaigns/get           — получить кампании
```
